OSI-lag Jobb Dataenhet Adressering Protokoller Hardware Teknologier
7. Applikasjonslaget Gir tjenester direkte til brukerapplikasjoner (nettleser, e-post, osv.) og sørger for at data kan sendes og mottas på en meningsfull måte. Data Ingen spesifikk adressering – bruker applikasjonsnavn eller URL. HTTP, HTTPS, FTP, SMTP, POP3, IMAP, DNS, DHCP Datamaskin, server, mobil, nettleser Web-tjenester, e-post, fjerninnlogging (SSH), API-er
6. Presentasjonslaget Oversetter, krypterer og komprimerer data slik at applikasjonslaget forstår det. Sikrer riktig format mellom avsender og mottaker. Data Ingen SSL/TLS, JPEG, MPEG, ASCII, GIF Server, datamaskin, applikasjonsprogramvare Kryptering (TLS/SSL), datakomprimering, formatkonvertering
5. Sesjonslaget Etablerer, styrer og avslutter kommunikasjonssessioner mellom applikasjoner. Håndterer dialogkontroll og synkronisering. Data Ingen RPC, NetBIOS, PPTP Server, applikasjonsprogramvare Fjernpålogging, filoverføring, videochat
4. Transportlaget Sørger for pålitelig (eller upålitelig) overføring av data mellom endepunkter. Deler opp data i segmenter og gjenoppretter dem ved mottak. Segment (TCP) / Datagram (UDP) Portnummer TCP, UDP Gateway, nettverkskort Streaming, nettspill, webtrafikk, e-post
3. Nettverkslaget Ruter datapakker gjennom nettverket og bestemmer den beste veien fra avsender til mottaker. Pakke IP-adresse IP, ICMP, ARP, RIP, OSPF Router, Layer 3-switch IPv4, IPv6, NAT, rutingprotokoller
2. Datalink-laget Sørger for feildeteksjon og -korreksjon på det lokale nettverket. Overfører rammer (frames) mellom noder på samme fysiske nettverk. Ramme (Frame) MAC-adresse Ethernet, PPP, ARP, VLAN (802.1Q) Switch, nettverkskort (NIC), bridge Wi-Fi (802.11), Ethernet (802.3), Bluetooth
1. Fysiske laget Overfører rå bits som elektriske, optiske eller radiobølger. Definerer kabler, signalstyrke og standarder for fysisk tilkobling. Bit Ingen Ethernet (fysisk), DSL, ISDN Kabler, hub, repeater, modem, nettverkskort Fiberoptikk, koaksialkabel, UTP, 4G/5G
Viktige protokoll-headere
Protokoll Viktige felt i headeren
Ethernet-header
  • Destination MAC-adresse
  • Source MAC-adresse
  • Type (identifiserer neste protokoll, f.eks. IPv4 = 0x0800)
IP-pakke (IPv4)
  • Versjon (IPv4/IPv6)
  • Header Length
  • Time To Live (TTL)
  • Protocol (f.eks. TCP=6, UDP=17)
  • Source IP / Destination IP
TCP-segment
  • Source Port / Destination Port
  • Sequence Number / Acknowledgment Number
  • Flags (SYN, ACK, FIN, RST)
  • Checksum
UDP-segment
  • Source Port / Destination Port
  • Length
  • Checksum
